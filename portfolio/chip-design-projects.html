FPGA RISC-V Processor

A RISC-V processor with a 5 stage pipeline and a L1 4 way cache. Follows the RISC-V ISA and can run assembly programs simulated on a BASYS3 development board. Implemented in Verilog and SystemVerilog

A RISC-V processors that follows modern computer architecture. Although modern chips usually have way more cores, pipelines up to 20 cycles long, and a lot of parallel distributions, this chip is almost there. It has a branch predictor, hazard detection unit, made for low power processing and all done in SystemVerilog and Verilog.

Blackjack

Blackjack ran on hardware. With a secret. Implemented all in Verilog and SystemVerilog

Although it probably would've been smarter to make the entire thing on the processor with assembly, I challenged myself to write the entire thing with an FPGA. The entire system communicates the game state to a VGA controller which then shows the entire game on a monitor via a VGA cable. This was part of a hardware security project so to stay on topic, there is a hardware trojan in the code. All that can be said about it is it rigs the game in your favor.

Ring Oscillator, Physically Unclonable Function

A PUF is a hardware mechanism used to create challenge and response pairs to verify on the hardware level if a device is who they say they are. Implemented with VHDL and SystemVerilog

The challenge here was the platform developed- Vivado, which did not like combinatorial loops. The entire RO PUF is an implementation of a research paper that talks about creating a ring oscillator as a PUF on artix FPGA boards, which was what the result was simulated on.

Pseudo RNG Accelerator

A low power efficient warbler RNG chip built with open source ASIC development software. IO via

Read a lot of RNG chip research papers and refitted a chip used on IR scanning stuff for use on cryptographic key generations. It stores the random numbers on a dual port ram implemented with the wishbone protocol. The RNG bitstreams will randomly change the RAMs while other cryptographic machines can use the other port to access and use the random numbers for whatever purpose they need.

Wishbone Dualport RAM

Dualport ram following wishbone protocol and architecture, built using defined macros and simulated with gate level testing.

RAM cannot be made and simulated because if it was, the amount of flip flops would crash computers. Instead, defined macros are used instead with realistic physics timings and more.
